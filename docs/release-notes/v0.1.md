# v0.1, initial capability portrait

**Type:** new repository (capability portrait #8, the regulated-reporting / CDx axis)

## What this is

A clean-room POC of the regulated-reporting axis of clinical bioinformatics,
built from public patterns on synthetic data: a vendor-agnostic CDx integration
shim, a clinical report generator with a content hash, and 21 CFR Part 11-style
electronic-signature controls with tamper detection. It closes the portfolio gap
between the "regulated clinical bioinformatics + CDx" positioning and public
proof (see the portfolio gap analysis).

## Shipped in v0.1

- `vendors.py`, three synthetic vendor schemas (Roche- / Thermo- / Guardant-
  shaped) + a canonical normalizer reconciling field names, pos/neg encodings,
  and VAF fraction-vs-percent.
- `report.py`, structured, versioned clinical report with per-finding therapy
  implications, QC status, and a canonical content SHA-256.
- `compliance.py`, electronic-signature records bound to the content hash, a
  hash-chained signature ledger, required-sign-off check, and re-verification
  that detects post-signature edits to the report or the chain.
- `pipeline.py`, CLI wiring vendor batch → normalize → report → sign → verify,
  with audit + MLflow hooks; writes a signed-record artifact.
- Substrate hooks (`audit`, `tracking`, `canary`), CI (ruff + pytest +
  scope-preamble + canary + English-only), MIT license, Makefile.

## Demo (synthetic, deterministic)

3-vendor batch → 6 canonical findings (EGFR / TP53 / KRAS / BRAF / ERBB2) →
report with authored + approved sign-offs → verification ok. The canary tampers
with a signed report and asserts verification fails. 18 tests, ruff + English-
only green.

## Scope

Synthetic only; the pattern, not a validated system; not regulatory advice. See
`what-is-out-of-scope.md`.
